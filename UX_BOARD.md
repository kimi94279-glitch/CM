# UX_BOARD.md

# Couple Map — Board & Place 경험 UX/UI 설계

## Overview

이 문서는 Couple Map의 핵심 경험인 Board/Place의 UX/UI를 정의한다.

- 대상: BoardList · CreateBoard · BoardDetail · PlaceSearch · PlaceAdd · PlaceCard · Empty States · Reorder UX
- RN 코드, Supabase 코드, 네이버 지도 SDK / 장소 검색 API 연동, 실제 UI 구현은 이 문서의 범위가 아니다.
- 시각 언어는 DESIGN.md, 데이터는 DATABASE.md, 흐름 정합은 FLOW_AUTH.md / UX_AUTH.md를 따른다.

### 반영된 결정

1. **BoardDetail 지도:** 30% 비율을 기본으로 하되 **접기 가능(collapsible)** 구조. 펼침 30% / 접힘 10%.
2. **PlaceAdd:** 추가 시점에 메모를 받지 않는다. "검색 → 선택 → 추가"로 끝낸다. 메모는 PlaceCard에서 작성.
3. **BoardList:** 감정 요소를 추가해 프로젝트 목록이 아닌 데이트 보드처럼 보이게 한다.
4. **PlaceCard:** 메모 없음 시 빈 공간 대신 자연스러운 권유형 CTA.
5. **Reorder:** 드래그 핸들 기반만 채택. 위/아래 버튼은 MVP 제외(접근성/Future).
6. **Core Experience:** BoardDetail을 핵심 경험으로 명시.

---

## 0. Core Experience (핵심 경험)

> BoardDetail은 단순 장소 목록이 아니다.

사용자는 BoardDetail에서

- 장소를 추가하고
- 메모를 남기고
- 반응을 남기고
- 순서를 조정하며

**하나의 데이트 코스를 둘이 함께 완성한다.**

이것이 Couple Map의 핵심 가치이며, 모든 Board/Place 설계는 이 경험을 강화하는 방향으로 판단한다.
지도는 보조 도구이고, 장소 카드와 협업 흔적(작성자·반응·메모)이 주인공이다.

## 2. 사용자 행동

| 화면 | 핵심 행동 |

## 3. 화면별 목적

- **BoardList:** "우리가 함께 만든 계획"을 데이트 보드로 보여준다. 업무 목록처럼 보이지 않게 감정 요소를 더한다.
- **CreateBoard:** 최소 마찰로 새 계획 시작(제목 한 줄).
- **BoardDetail:** 코스를 함께 완성하는 핵심 공간. 지도는 접을 수 있는 보조, 장소 카드가 주인공.
- **PlaceSearch:** 가고 싶은 장소를 빠르게 찾는다.
- **PlaceAdd:** 군더더기 없이 빠르게 장소를 보드에 담는다.
- **PlaceCard:** 장소명·메모·반응·작성자를 한 장에서 확인(협업 흔적의 단위).
- **Empty States:** 다음에 무엇을 할지 자연스럽게 안내한다.
- **Reorder UX:** 코스 순서를 직관적으로 조정한다.

## 4. 와이어프레임 (텍스트)

### ① BoardList (감정 요소 반영)

```text
 우리의 데이트                 🟠🟢
 ─────────────────────────────
 ┌───────────────────────────┐
 │ ❤ 토요일 성수 데이트         │
 │ 📍 장소 5개                 │
 │ ☕ 최근: 카페 어니언          │  ← 마지막으로 추가한 장소 미리보기
 └───────────────────────────┘
 ┌───────────────────────────┐
 │ 🌸 부산 여행                │
 │ 📍 장소 8개                 │
 │ 📸 최근: 감천문화마을         │
 └───────────────────────────┘
                          ( + )  ← 새 보드 FAB
```

- "마지막 수정 시간" 대신 **"최근 추가한 장소"**를 노출해 데이트 보드의 정서를 살린다.
- 각 보드에 감정 이모지/색을 부여(생산성 목록 느낌 제거).

### ② CreateBoard

```text
 ← 새 데이트 계획
   어떤 데이트를 계획해볼까요?

 [ 예: 토요일 성수 데이트          ]

 [          만들기          ]   ← Primary (제목 입력 시 활성)
```

### ③ BoardDetail — 핵심 화면 (지도 접기 가능)

**기본 상태 (지도 30% / 카드 70%)**

```text
 ← 토요일 성수 데이트                 ⋯
 ┌──── 지도 (약 30%) ───────  [▲ 접기] ┐
 │   ①─②─③  핀 + 연결선                │
 └──────────────────────────────────────┘
 계획 (약 70%, 스크롤)
 ┌───────────────────────────┐
 │ ① 카페 어니언          🟠  │
 │   "디저트 맛있대"           │
 │   [❤ 2] [☕ 1]          ⠿ │  ← PlaceCard
 └───────────────────────────┘
 ┌───────────────────────────┐
 │ ② 성수연방            🟢  │
 │   + 왜 가고 싶은지 적어볼까요? │
 │   [📸 1]                ⠿ │
 └───────────────────────────┘
                        ( + 장소 )  ← FAB
```

**접은 상태 (지도 10% / 카드 90%)**

```text
 ← 토요일 성수 데이트                 ⋯
 ┌─ 지도 (약 10%) ─────────  [▼ 펼치기] ┐
 └──────────────────────────────────────┘
 계획 (약 90%, 스크롤)
 ┌───────────────────────────┐
 │ ① 카페 어니언 …             │
 ... (카드에 집중)
```

- 기본 30%, 접으면 10%. 사용자가 필요할 때만 펼친다.
- Couple Map은 지도 앱이 아니라 계획 앱이므로, 카드에 머무는 시간을 우선한다.

### ④ PlaceSearch

```text
 ← 장소 추가
 [ 🔍 장소를 검색해보세요            ]
 ─ 결과 ─
 │ 카페 어니언    성동구 아차산로…  [+] │
 │ 성수연방       성동구 …         [+] │
```

### ⑤ PlaceAdd (메모 없음 — 빠른 추가)

```text
 카페 어니언
 성동구 아차산로…

 [        이 장소 추가하기        ]   ← Primary
```

- 흐름: **검색 → 선택 → 추가**. 메모 입력 단계 없음.
- 추가 후 PlaceCard가 생성되고, 메모는 카드에서 작성한다.

### ⑥ PlaceCard (핵심 컴포넌트)

**메모 있음**

```text
 ┌──────────────────────────────┐
 │ ①  카페 어니언          🟠   │  번호=순서 / 점=작성자(You/Partner)
 │    "디저트 맛있대"            │
 │    [❤ 2] [☕ 1] [📸] [🌸]  ⠿ │  채워진 칩=내가 누른 반응 / ⠿=드래그 핸들
 └──────────────────────────────┘
```

**메모 없음 (빈 공간 대신 권유형 CTA)**

```text
 ┌──────────────────────────────┐
 │ ②  성수연방             🟢   │
 │    + 왜 가고 싶은지 적어볼까요? │  ← 자연스러운 메모 유도
 │    [❤] [☕] [📸] [🌸]      ⠿ │
 └──────────────────────────────┘
```

- 작성자 색 점(🟠 You / 🟢 Partner)으로 누가 추가했는지 즉시 식별.
- 반응 칩은 고정 4종(❤️ 가고 싶어 / ☕ 카페 좋아 / 📸 사진 / 🌸 분위기), 채워짐 = 내가 누른 상태 + 카운트.

## 6. 빈 상태 (Empty States)

| 상황 | 화면 | 카피 · 안내 |
| --- | --- | --- |
| 첫 보드 없음 | BoardList | "첫 데이트 계획을 함께 만들어볼까요?" + 큰 [+ 새 보드]. 온보딩 직후 첫 화면이므로 가장 중요. |
| 장소 없음 | BoardDetail | "가고 싶은 곳을 추가해보세요" + [+ 장소] 강조. 지도는 빈 안내 상태. |
| 검색 결과 없음 | PlaceSearch | "검색 결과가 없어요. 다른 키워드로 찾아보세요" |

- Empty State는 "비어 있음"을 보여주는 것이 아니라 **다음 행동을 권유**하는 화면이다.

## 8. Reorder UX (드래그 기반)

### 채택안

- **드래그 핸들(⠿)을 길게 눌러 정렬**을 유일한 방식으로 채택한다.
- 드래그 중 햅틱 피드백, 지도 핀 번호와 카드 순서를 동기화한다.
- 위/아래 이동 버튼, 번호 직접 입력은 MVP에서 제외한다(접근성/Future로만 기록).

### 데이터 정합

- DATABASE.md의 `places (board_id, order_index)` UNIQUE 제약과 정합하도록 **재정렬은 트랜잭션 일괄 갱신**으로 저장한다(드롭 시 한 번에 반영, 실패 시 전체 롤백).

### 협업 동시성

- 두 사람이 동시에 정렬할 수 있다. 저장 충돌 시 마지막 저장을 반영하고, 충돌한 쪽은 롤백 후 재시도한다.
- Realtime은 ARCHITECTURE.md의 범위(장소 추가/삭제/순서 변경/메모/반응)에 한정한다.

### 분석 요약 (왜 드래그인가)

| 방식 | 장점 | 단점 | 결정 |
| --- | --- | --- | --- |
| 드래그 핸들 | 직관적, 코스 감각, DESIGN과 일치 | 긴 목록 스크롤과 제스처 충돌 가능 | **채택** |
| 위/아래 버튼 | 단순, 접근성 우수 | 여러 칸 이동 번거로움, 카드 복잡화 | Future(접근성) |
| 번호 직접 입력 | 정밀 | 데이트 정서와 불일치, 부담 | 제외 |

## 9. 협업 흔적 가시성 (Collaboration Visibility)

Couple Map은 협업 서비스다. 다음을 항상 쉽게 확인할 수 있어야 한다.

- **누가 추가했는지:** PlaceCard의 작성자 색 점(You/Partner).
- **누가 반응했는지:** 반응 칩의 채워짐/카운트(내가 누른 반응 강조).
- **어떤 메모를 남겼는지:** 카드 본문 메모, 없으면 권유형 CTA.

혼자 쓰는 듯한 화면이 되지 않도록, 두 사람의 흔적을 시각적으로 노출하는 것을 기본으로 한다.

## 10. Future Considerations

현재 MVP 범위가 아니며, 향후 검토 대상으로만 기록한다.

- **지도 협업 캔버스 (장기 비전):** 사용자가 지도 위에 직접 **손글씨 · 화살표 · 도형 · 텍스트 메모**를 남길 수 있는 **지도 + 화이트보드 형태의 공동 편집 기능**. 두 사람이 지도 위에 생각을 자유롭게 표현하는 경험. Couple Map의 장기 비전이며 MVP에는 포함하지 않는다.
- **Reorder 접근성:** 위/아래 이동 버튼 등 드래그 대체 수단.
- **BoardCard 풍부화:** 대표 이미지/썸네일 등 감정 요소 강화(스토리지 도입 시).

## Amendment: Solo Mode (0002)

> "혼자 사용 모드 미제공" 전제는 폐기되었다. Board/Place 경험은 솔로 워크스페이스에서도 동일하게 동작하도록 설계한다(데이터는 워크스페이스 = couples 행에 소속).

- 보드/장소는 커플 연결 여부와 무관하게 워크스페이스(`couples`) 단위로 저장된다. 초대 수락 시 솔로 워크스페이스가 `active`로 승격되어 **기존 보드/장소가 그대로 유지**된다.
- **빈 상태 전환 CTA(예정):** BoardList/BoardDetail 빈 상태에 "함께하면 더 좋아요 · 상대 초대" 보조 CTA를 둔다(강제 아님).
- 협업 흔적(작성자 색 등)은 솔로에서는 1인만 표시되고, 연결 후 2인으로 자연스럽게 확장된다.
