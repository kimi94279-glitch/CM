# Couple Map Design Guide

## Purpose

이 문서는 Couple Map의 시각/경험 디자인 기준을 정의한다.

핵심 가치는 다음과 같다.

> 지도 앱이 아니라, 둘이 함께 데이트를 준비하는 공간.

모든 디자인 결정은 이 가치를 시각 언어로 번역하는 것을 목표로 한다.

> 실제 UI 코드, 컴포넌트 코드, 디자인 토큰 코드화, 에셋 제작은 이 문서의 범위가 아니다.

---

# 0. 디자인 철학

사용자는 계획을 세우려 앱을 열지만, 결과적으로 함께 웃고 상상하고 기대하는 경험을 해야 한다.

- **지도는 주인공이다** — 모든 경험은 지도 위에서 이해될 수 있어야 한다.
- **계획은 놀이처럼** — 장소를 추가·정렬·연결하는 과정이 업무가 아닌 가벼운 놀이로 느껴져야 한다.
- **커플 앱처럼 보이지 않는다** — 하트·분홍 장식이 아니라 함께 계획하는 행동에 집중한다.
- **복잡함보다 직관** — 지도를 보는 순간 "여기 가려고 모아둔 곳들이구나"가 이해돼야 한다.
- **계획 자체가 추억이다** — 방문 이후보다 방문 이전의 기대감을 강화한다.

피해야 할 방향: 생산성 툴 · 여행 일정표 · 커플 SNS · 과도하게 감성적인 디자인 · 과다한 정보 표시.

# 1. 디자인 컨셉

## 컨셉 키워드

**"함께 그리는 데이트 노트" (A date note we draw together)**

- 지도 도구가 아니라 둘이 채워가는 노트/보드의 감각.
- 차갑고 정밀한 내비게이션 UI가 아니라, 따뜻하고 가벼운 종이/카드 질감.
- 핵심 무드: 부드러움 · 설렘 · 가벼움 · 둘이라는 느낌

## 안티 패턴

- 대시보드
- 표·그리드 위주의 생산성 툴
- 길찾기 중심의 풀스크린 지도

## 원칙 번역

| 원칙(CLAUDE.md) | 디자인 번역 |
| --- | --- |
| 계획이 중심 | 지도는 화면의 보조 영역, 장소 카드 리스트가 주인공 |
| 협업이 중심 | 두 사람의 흔적(메모·반응·작성자)을 항상 시각적으로 노출 |
| 가벼운 상호작용 | 탭 한 번의 반응(이모지), 짧은 메모, 드래그 정렬 |
| 단순함 | 한 화면에 하나의 핵심 행동 |

# 2. 브랜드 방향성

## 퍼스낼리티

다정한 친구 같은 톤.

명령형 대신 권유형 카피를 사용한다.

예시

- "어디 가고 싶어?"
- "장소 추가해볼까?"

## 톤 & 보이스

- 짧고 구어체
- 이모지 절제 사용
- 부담 없는 안내

## 두 사람성 (Two-ness)

커플 각자에게 고정 개인 색(You / Partner)을 부여한다.

누구의 제안·메모·반응인지 한눈에 구분할 수 있어야 한다.

## 로고/네이밍 무드

둥근 핀 두 개가 겹친 형태.

두 사람이 한 장소에서 만나는 은유.

> 시각 컨셉만 제안하며, 실제 제작은 범위 밖이다.

# 3. 컬러 시스템

MVP 단순성을 위해 토큰을 최소로 유지한다.

> 실제 코드화(디자인 토큰 작성)는 범위 밖이다.

## Primary / Brand

- `Primary` — Coral Pink `#FF6B81` : 설렘, 주요 액션(저장/추가)
- `Primary Soft` — `#FFE3E8` : 선택 강조 배경, 핀 강조

## Couple Identity (협업의 핵심)

- `You` — Warm Coral `#FF6B81`
- `Partner` — Calm Teal `#3BC9B0`

두 색은 메모/반응/작성자 표시에 일관 적용한다.

## Neutral (종이 느낌)

- `BG` `#FFFBF7` (살짝 따뜻한 화이트)
- `Surface` `#FFFFFF`
- `Text Strong` `#2B2B2E`
- `Text Muted` `#8A8A8F`
- `Border` `#EFE9E3`

## Semantic

- `Success` `#3BC9B0`
- `Warning` `#FFB020`
- `Danger` `#FF5A5A`

## 원칙

채도 높은 색은 액션·정체성에만 사용한다.

배경은 무채색에 가까운 따뜻한 톤으로 차분하게 유지한다.

# 4. 타이포그래피

## 서체

시스템 폰트 우선.

- iOS: San Francisco
- Android: Roboto
- 한글: 시스템 기본 (Apple SD Gothic / Noto Sans KR)

> React Native 기본 폰트를 사용해 설치 부담을 0으로 유지한다.

## 스케일 (모바일 기준)

| 토큰 | 크기/굵기 | 용도 |
| --- | --- | --- |
| Display | 28 / Bold | 보드 제목, 환영 화면 |
| Title | 20 / SemiBold | 화면 헤더 |
| Body | 16 / Regular | 장소명, 본문 |
| Label | 14 / Medium | 메타 정보, 버튼 |
| Caption | 12 / Regular | 작성자, 시간, 도보 시간 |

## 규칙

- 줄간격 1.4
- 한 줄 텍스트 우선 (카드 안에서 말줄임 처리)

# 5. 컴포넌트 스타일

## 공통 스타일

- 둥근 모서리 (radius 16)
- 부드러운 그림자 (낮은 elevation)
- 넉넉한 패딩
- 종이 카드 질감

## PlaceCard (핵심 컴포넌트)

- 구성: 장소명(Body) · 짧은 메모 · 반응 칩 · 작성자 점(You/Partner 색) · 드래그 핸들
- 순서(order_index)를 좌측 번호 뱃지로 표시하여 "코스" 느낌을 준다

## ReactionButton

- 알약형 칩
- 탭 시 색 채워짐 + 카운트
- 고정 4종 (❤️ / ☕ / 📸 / 🌸)

## MemoInput

- 한 줄 입력
- placeholder "메모 남기기…"
- 가벼운 인라인 저장

## BoardCard

- 보드 목록용 큰 카드
- 제목 + 장소 수 + 두 사람 아바타

## Button

- Primary (채움) / Secondary (외곽선) / Ghost (텍스트)
- radius 12
- 높이 48 (한 손 탭)

## FAB (+)

- 우하단 위치
- 장소 추가의 단일 주요 동선

## Avatar

- 닉네임 이니셜 + 개인 색 링

## ARCHITECTURE 매핑

이 컴포넌트들은 `src/components/`의 책임(PlaceCard, ReactionButton, BoardHeader 등)과 1:1 대응한다.

# 6. 화면 와이어프레임

MVP Screens(PRD) 기준. 레이아웃 흐름만 정의하며, 픽셀 디자인은 후속 작업이다.

## Board List (홈)

```text
[ 우리의 데이트 ▾ ]            (헤더, 두 사람 아바타)
─────────────────────────
│ ❤ 토요일 성수 데이트       │
│  장소 5 · 🟠🟢            │  ← BoardCard
─────────────────────────
│ 🌸 부산 여행              │
─────────────────────────
                       ( + )  ← 새 보드 FAB
```

## Board Detail (핵심 화면)

```text
← 토요일 성수 데이트
┌───── 지도 (보조, 상단 1/3) ─────┐
│  ①──②──③  연결선 + 핀          │
└────────────────────────────────┘
계획 ──────────────────  (주 영역)
① 카페 어니언   ☕x2 ❤x1   🟠
   "디저트 맛있대"            ⠿
② 성수연방      📸x1        🟢
   "사진 찍기 좋음"          ⠿
③ ...
                          ( + 장소 )
```

## Place Search / Add

```text
← 장소 추가
[ 🔍 장소 검색…              ]
─ 검색 결과 리스트 ─
│ 카페 어니언  성동구 …   [추가] │
─────────────────────────
(선택 시 → 보드에 카드로 삽입)
```

## Couple Invite / Join

```text
   둘이 함께 시작해요
   [ 초대 링크 복사 ]
   또는
   [ 초대 코드 입력 ____ ]
```

## Auth (Login / Signup)

```text
   Couple Map
   함께 만드는 데이트 계획
   [ Google로 계속하기 ]
   [ 이메일로 계속하기 ]
```
